How KickPick works

Three signals. One verdict.
Ad
📊

Live stats model

We pull today's fixture list from API-Football each morning. For every match, we collect average goals scored and conceded, possession stats, shot conversion, H2H results over the last 5 seasons, and recent form (last 6 games). A probability model converts these into 1/X/2 win percentages.

🤖

AI analysis

Each match's stat package is sent to the Claude API with a structured prompt. Claude analyses form, momentum, tactical matchups, and injury context to generate a prediction and confidence score. This layer catches things raw stats miss — a team on a 5-game winning streak but all in weak form, for instance.

👥

Tipster consensus

We scrape prediction percentages from Forebet, Betensured, and Windrawwin — three of the largest public prediction engines. We weight their consensus by their historical accuracy on that specific league and time of season. The result is a crowd-wisdom signal that complements the AI view.

⚡

The verdict

When all three signals align — stats model, AI, and tipster consensus — we flag it as a high-confidence pick. Divergence between signals is surfaced transparently so you can make your own call. We never manufacture agreement that isn't there.

Data sources

API-Football apifootball.com

Live fixture data, team stats, form, lineups, injuries, and bookmaker odds. Free tier covers 100 calls/day — enough for a daily cron job covering all major European leagues. Upgrade to paid for real-time livescores.

Claude API (Anthropic) anthropic.com

Each match gets a structured prompt including H2H records, recent form, and statistical context. Claude returns a JSON object with a prediction, confidence score, and two-sentence reasoning. Runs via a Vercel serverless function, cached per match per day to control API costs.

Tipster aggregation Forebet · Betensured · Windrawwin

A lightweight Python scraper (running as a Vercel cron) collects the 1/X/2 consensus percentages from three public tipster engines each morning. Results are averaged and stored in a Vercel KV (Redis) store. Response times under 50ms.

Technical architecture

Deployment stack Vercel · GitHub

The site is a static HTML/CSS/JS frontend deployed on Vercel — identical architecture to MiaPensione. Data is fetched and cached by two serverless functions:

  • /api/matches.js — calls API-Football, runs Claude picks, returns enriched fixture JSON
  • /api/tipsters.js — scrapes consensus data, merges with match data

Both functions run once daily at 07:00 UTC via Vercel Cron. Results are stored in Vercel KV so subsequent page loads serve cached data instantly — no runtime cost per visitor.
